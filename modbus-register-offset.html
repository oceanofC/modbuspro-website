Modbus Register Offset: The 0-Based vs 1-Based Problem

TROUBLESHOOTINGUpdated Feb 20268 min read

Modbus register addressing is one of the most confusing aspects of the protocol â€” not because it's complicated, but because there are three different conventions used by documentation, tools, and the actual wire protocol, and they all disagree by 1.

Why It's Confusing

Imagine a device with holding registers starting at "address 1" according to its manual. Depending on which Modbus tool you use and how it's configured, you might need to enter 0, 1, 40001, or 40000 to read that same register. All four refer to the same physical data location. This isn't a bug â€” it's three overlapping conventions from the 1970s that refuse to die.

The Wire Protocol vs Documentation

On the wire, Modbus always uses 0-based addressing. When your master sends a request for "address 0", it gets the first register. Address 1 gets the second register. This is the absolute truth â€” it's what the bytes in the frame contain.

But many manufacturer manuals use 1-based numbering. They call the first register "register 1". So when you see "Register 1" in a datasheet, you need to send address 0 in your Modbus frame.

The 4xxxx / 3xxxx Prefix Convention

The original Modicon convention adds a prefix digit to indicate the register type:

  • 0xxxx â€” Coils (FC01/05)
  • 1xxxx â€” Discrete Inputs (FC02)
  • 3xxxx â€” Input Registers (FC04)
  • 4xxxx â€” Holding Registers (FC03)

So "40001" means "the first holding register". To get the wire address: strip the prefix digit, subtract 1. 40001 â†’ 0001 â†’ 0000. "40100" â†’ 0100 â†’ 0099.

How Different Tools Handle It

This is where it gets really fun. Different Modbus tools handle the offset differently:

  • Modbus Poll â€” 0-based by default (matches wire protocol)
  • Ignition Modbus driver â€” Configurable, but default behaviour depends on the address format used
  • ModRSsim2 â€” 0-based (matches wire)
  • PLCnext Modbus function blocks â€” 0-based (matches wire)
  • Many SCADA systems â€” 1-based with "PLC address" option

Always check your tool's documentation for whether it expects 0-based or 1-based addresses.
