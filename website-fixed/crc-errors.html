CRC errors mean data is arriving but it's corrupted in transit. The message integrity check is failing. Here's what causes it and how to eliminate it.

What Is a CRC Error?

Every Modbus RTU frame ends with a 16-bit Cyclic Redundancy Check. The sender calculates a checksum from the message bytes and appends it. The receiver calculates the same checksum on the received data â€” if they don't match, the frame is discarded. Unlike "no response," a CRC error means data IS flowing, but it's being corrupted.

CRC errors only apply to Modbus RTU (serial). Modbus TCP uses TCP's own error detection and doesn't include a Modbus-level CRC. If you're seeing data corruption on TCP, the problem is at a higher level.

Common Causes

1. Electrical Noise / EMI

The #1 cause in industrial environments. RS-485 cables running alongside VFD power cables, near switchgear, or through electrically noisy environments pick up interference that flips bits in the data stream.

Fixes:

  • Use shielded twisted pair cable â€” the twist rejects common-mode noise, the shield blocks electric field interference
  • Route data cables separately from power cables â€” minimum 300mm separation, cross at 90Â° if they must
  • Ground the shield at one end only to avoid ground loops
  • Add ferrite cores near the connection points

2. Baud Rate or Framing Mismatch

If the master and slave have slightly different communication parameters, frames arrive but decode incorrectly. Even a parity bit mismatch will cause consistent CRC failures â€” the bytes shift and the CRC never matches.

Fix: Triple-check baud rate, data bits, parity, and stop bits on both ends.

3. Missing or Incorrect Termination

Without 120Î© termination resistors at both ends of the RS-485 bus, signal reflections occur. At higher baud rates or longer cable runs, these reflections corrupt the waveform enough to cause bit errors.

Fix: Install 120Î© resistors between the A and B lines at the first and last device on the bus. Remove any termination from intermediate devices.

4. Bus Contention

If two masters are accidentally connected to the same RS-485 bus, or a slave starts transmitting before the previous slave finishes, the signals collide and both frames are corrupted.

Fix: Verify there is only one master on the bus. Check for devices with firmware bugs that cause late or overlapping responses.

5. Cable Length Exceeded

RS-485 can theoretically run 1200m, but this depends on baud rate. At 115200 baud, reliable distance drops to ~100-200m. Signal attenuation over long runs reduces the voltage margin until the receiver can no longer distinguish clean data.

Fix: Reduce baud rate for long distances, or add RS-485 repeaters to extend range.
