The Problem

Modbus has a legacy numbering system from the original Modicon PLCs that adds a prefix to indicate the register type, and uses 1-based numbering. But the actual protocol on the wire uses 0-based addressing with no prefix. The result: the same register can be described as "40001" in documentation and as address "0" in the Modbus frame.

Some documentation uses 0-based numbering without the prefix. Some uses 1-based WITH the prefix. Some uses 1-based WITHOUT the prefix. You need to determine which convention your specific documentation uses, or you'll be permanently off by one on every register.

How to Tell Which Convention a Document Uses

  1. Look for numbers starting with 3xxxx or 4xxxx â€” if you see "40001", it's the Modicon convention. Subtract the prefix (40001) to get wire address 0.
  2. Look for "register 1" as the first register â€” if the first register is listed as "1" (not "0"), it's 1-based. Subtract 1 for the wire address.
  3. Look for "register 0" or "address 0" â€” this is the actual wire address format. No conversion needed.
  4. Check hex addresses â€” if the doc shows 0x0000 as the first address, it's 0-based wire addressing.

Real-World Example

ABB REX615 documentation lists "Voltage L1" at Modbus address 2999 (0-based, hex 0x0BB7). This is the actual wire address â€” type 2999 directly into your Modbus tool. No offset needed.

Some Schneider meters list "Voltage L1" at register 30001. This means Input Register at wire address 0, read with FC04.
