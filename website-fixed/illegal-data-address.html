Modbus Exception 02: Illegal Data Address

The most common exception code in practice. The slave is telling you: "that register doesn't exist here." Here's every reason why, and how to fix each one.

Troubleshooting Updated Feb 2026 7 min read

What Exception 02 Means

Exception code 02 (Illegal Data Address) tells you the slave received your request correctly and understood the function code, but the register address or address+quantity combination doesn't map to a valid data point on this device. The good news: your comms link is working perfectly.

Cause 1: Address Offset (0 vs 1)

The most frequent cause. See our dedicated guide on Register Address Offset, but in short: if documentation says "register 40001", the wire address is 0, not 40001.

Cause 2: Wrong Function Code

Register address 100 accessed via FC03 (Holding Registers) and register address 100 via FC04 (Input Registers) are completely different registers. The device may have data at address 100 in input registers but nothing at address 100 in holding registers.

Fix: Try switching between FC03 and FC04. Check the documentation for which function code maps to measurement data on your specific device.

Cause 3: Quantity Overflow

Your starting address is valid, but address + quantity extends past the end of the device's register map. Example: registers exist from 0-99, you request starting address 90 with quantity 20 ‚Äî address 100+ doesn't exist.

Fix: Reduce the quantity or split into multiple smaller reads. Try reading just 1 register first to confirm the starting address works.

Cause 4: Register Gaps

Not every address between the first and last register is necessarily valid. Some devices have non-contiguous register maps with gaps. If you try to read a block that spans a gap, the device returns exception 02 even though the start and end addresses both exist individually.

Fix: Split your read into smaller contiguous blocks that don't span gaps. Some devices with incomplete Modbus implementations are hardcoded to respond to specific request patterns ‚Äî try reading the exact blocks shown in the documentation examples.

Cause 5: Firmware Version Mismatch

Register maps evolve across firmware versions. A register added in firmware v3.0 won't exist on a device running v2.5. This is especially common with ABB and Schneider devices that get regular firmware updates adding new measurement parameters.

Fix: Check the device firmware version and match it to the correct documentation revision. Consider updating the firmware if the register you need was added in a newer version.

Cause 6: Device Configuration

Some registers only exist when certain features are enabled. For example, harmonics analysis registers may only be available when the harmonics module is enabled in the device configuration. Power factor registers may not exist on a device configured for voltage-only measurement.
